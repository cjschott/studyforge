# StudyForge v0.6.0-alpha.3 Publish Pipeline And Export Hardening Plan

## Objective

Make reviewed question publishing traceable, reversible, and exportable as a reusable course pack without adding real AI, embeddings, auto-publish behavior, or replacing SQLite.

## Scope

- Add publish history storage for publish, republish, retire, and restore actions.
- Add published-question lineage snapshots copied from draft lineage at publish time.
- Keep first publish creating a real `questions` row and later publishes updating that same row.
- Hide retired questions from normal study by default while keeping them visible in admin/review paths.
- Add question publish-history and lineage APIs.
- Add course export options for retired questions, draft inclusion, lineage, and review metadata.
- Add export validation warnings for missing lineage, unpublished verified drafts, unreviewed questions, unresolved high-severity conflicts, and rejected concepts linked to published questions.
- Update Question Drafts, Admin review/export, and Course Builder UI to expose the new publish/export state.

## Verification

- Backend tests cover publish history, lineage snapshots, high-severity publish blocking, retire/restore visibility, export options, and export validation.
- Frontend tests cover publish history rendering, retire/restore calls, lineage viewer rendering, export option query params, and Course Builder export readiness.
