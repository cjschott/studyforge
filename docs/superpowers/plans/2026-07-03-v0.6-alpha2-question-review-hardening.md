# StudyForge v0.6.0-alpha.2 Question Review Hardening Plan

## Objective

Harden question draft review so every publishable draft has persisted validation warnings, source/concept lineage checks, explanation quality metadata, and explicit human review gates.

## Scope

- Add `question_draft_warnings` and `QuestionDraft.explanation_json`.
- Add `question_validation_service` for structured warnings.
- Refresh warnings on draft create, save, validate, review, verify, reject, and publish attempts.
- Allow review/reject with warnings.
- Block verify/publish when high-severity warnings exist.
- Preserve first-publish/create and later-publish/update behavior through `published_question_id`.
- Extend Question Drafts UI with warning groups, high-severity filters, lineage metadata, and structured explanation editing.
- Extend Course Builder selected-source stats with warning, ready-to-publish, and published draft counts.

## Out of Scope

- Real LLM providers.
- Vector DB or embeddings.
- Auto-publishing generated content.
- Question generation beyond the existing rule-based draft foundation.
- SQLite replacement.

## Verification

- Backend question draft tests cover validation, duplicate warnings, refresh behavior, high-severity gates, and idempotent publish updates.
- Frontend tests cover warning rendering/filtering, structured explanation payloads, ready-to-publish state, and Course Builder warning stats.
