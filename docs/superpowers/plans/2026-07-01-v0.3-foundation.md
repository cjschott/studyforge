# StudyForge v0.3 Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a FastAPI/SQLite backend foundation while preserving StudyForge's existing static JSON/localStorage mode.

**Architecture:** Keep the vanilla frontend as the primary shell and add optional API adapters that activate only when `/api/admin/health` is reachable and the user is authenticated. The backend owns durable course data, users, auth, imports/exports, question workflow states, and cross-device progress; static JSON export remains compatible with the current frontend.

**Tech Stack:** Python FastAPI, SQLAlchemy ORM, SQLite, passlib bcrypt, PyJWT, pytest/TestClient, plain HTML/CSS/vanilla JS.
