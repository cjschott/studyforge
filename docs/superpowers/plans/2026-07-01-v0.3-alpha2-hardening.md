# StudyForge v0.3 Alpha.2 Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Harden the v0.3 foundation into a deployable alpha with usable admin workflows, progress sync, question review, source lineage visibility, expanded Security+ data, and schweb2 deployment docs.

**Architecture:** Keep the existing FastAPI/SQLite backend and vanilla frontend. Add small backend helper functions for operational config and response consistency, expand existing routers instead of replacing them, and keep frontend backend mode optional with localStorage as immediate cache.

**Tech Stack:** FastAPI, SQLAlchemy, SQLite, passlib bcrypt, PyJWT, pytest/TestClient, vanilla JavaScript, static JSON course packs.
