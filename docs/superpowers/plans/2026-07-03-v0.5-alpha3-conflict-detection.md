# StudyForge v0.5.0-alpha.3 Conflict Detection Plan

## Objective

Add a non-AI conflict detection and source validation foundation that stores findings, exposes a review UI, and warns Course Builder users about unresolved high-severity conflicts.

## Backend

- [x] Add `source_conflicts` model with source/chunk/concept lineage, evidence snippets, conflict type, severity, status, detection method, and timestamps.
- [x] Add authenticated conflict list/detail/update/resolve APIs.
- [x] Add rule-based source-material conflict detection for legacy exam references, low-authority/unverified sources, and answer-key-like chunks.
- [x] Add rule-based concept conflict detection for duplicate-looking concepts, missing lineage, authority spread, and verified concepts linked to weak source evidence.
- [x] Include source-material unresolved conflict stats for Source Library and Course Builder consumers.

## Frontend

- [x] Add Conflicts sidebar section for authenticated users.
- [x] Add conflict list filters for severity, status, type, and search.
- [x] Add conflict detail evidence view and review/resolve/reject controls.
- [x] Add Detect Conflicts actions and related conflict sections to Source Material and Concept detail views.
- [x] Add Course Builder unresolved conflict counts and high-severity warning.

## Verification

- [x] Add backend conflict persistence, filtering, detection, status, and stats tests.
- [x] Add frontend conflict list/filter/detail/action tests.
- [x] Extend Course Builder tests for conflict aggregation and high-severity warnings.
