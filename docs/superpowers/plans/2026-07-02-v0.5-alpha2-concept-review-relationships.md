# StudyForge v0.5.0-alpha.2 Concept Review And Relationship Mapping

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Improve the concept system so StudyForge can review concepts, manage aliases, merge duplicates, preserve source lineage, and map relationships without adding AI generation, embeddings, vector search, or replacing SQLite.

**Architecture:** Extend the existing v0.5 concept router and frontend module rather than creating a new subsystem. Keep source chunks and source-concept links as the lineage backbone, use soft rejection for merged concepts, and keep Course Builder as a read-only selected-context summary.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, SQLite, vanilla JavaScript modules, Node test runner, pytest.
