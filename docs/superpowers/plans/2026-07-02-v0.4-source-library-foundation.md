# StudyForge v0.4 Source Library Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the authenticated Source Library foundation for safely storing source materials, extracting text, chunking it, and previewing chunks before future AI ingestion.

**Architecture:** Keep the existing FastAPI + SQLite backend as the source of truth. Add source-library ORM models, Pydantic schemas, file/extraction/chunking services, and a `source_libraries` router under `/api`. Add a backend-only upload directory at `backend/data/sources/originals/` so uploaded files are not served by the static frontend. Add simple vanilla JS screens that appear only in backend mode.

**Tech Stack:** FastAPI, SQLAlchemy, SQLite, Pydantic v2, python-multipart, python-docx, pypdf, vanilla JS modules, Node `node:test`, pytest.
