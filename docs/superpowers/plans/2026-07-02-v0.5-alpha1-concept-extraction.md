# StudyForge v0.5.0-alpha.1 Concept Extraction Foundation

**Goal:** Extract, store, review, and manage concepts from existing Source Library chunks without adding Ollama, real LLM calls, question generation, embeddings, or a vector database.

## Scope

- Extend the concept data model with normalized names, description, course code, status, confidence, aliases, source lineage, evidence text, extraction method, and typed relationships.
- Add authenticated concept APIs and admin-only delete semantics that preserve source lineage by rejecting concepts instead of removing linked records.
- Add a rule-based extractor for headings, glossary-like lines, repeated capitalized terms, and a small Security+/networking keyword list.
- Add Concepts frontend list/detail screens, Source Material concept extraction controls, and Course Builder concept counts.
- Add backend and frontend tests for concept extraction, deduplication, lineage, review status, filtering, and UI helpers.
